

# Технологічний стек
**Frontend**: Next.js 16.1.6, React 19.2.3, TypeScript
**Стилізація**: SCSS, CSS Modules
**Форми**: React Hook Form + Zod валідація
**Додаткові**: ClassNames для умовних класів

## Пошукова система
- Форма з валідацією через Zod
- React Hook Form для керування формами
- Fake API для демонстрації


# Встановлення залежностей 
    npm install
# Запуск dev сервера 
    npm run dev
# Білд проекту 
    npm run build


## Принципи архітектури:
1. Separation of Concerns - кожна папка має свою відповідальність
2. Reusability - переферійні компоненти в _ui та _components
3. Feature-based - модулі організовані за функціональністю
4. DRY - загальна логіка винесена в хуки та утиліти

## Модульна архітектура Проект використовує модульну архітектуру для кращої організації коду: 
### Переваги модульної структури: 
- **Масштабованість**: Легко додавати нові модулі/сторінки 
- **Підтримуваність**: Кожен модуль відповідає за конкретну функціональність 
- **Переферійні залежності**: Загальні компоненти винесені в окремі директорії 
- **Ізоляція**: Модулі не залежать один від одного 

### Структура проекту: 

``` 
├──src/app/ 
    ├── _assets/ # Глобальні стилі і змінні SCSS 
    ├── _components/ # Переферійні компоненти (Header, Categories) 
    ├── _modules/ # Модулі сторінок (Home, CategoryPage) 
    ├── _ui/ # UI компоненти (H1, CenteredContainer) 
    ├── _hooks/ # Кастомні React хуки 
    ├── _types/ # TypeScript типи 
    ├── _utils/ # Допоміжні функції
```
